# MediChat Pro

A simple RAG (Retrieval-Augmented Generation) chatbot that lets you upload medical PDF documents and ask questions about them. Built with Streamlit, LangChain, FAISS, and Groq.

## Features

- Upload one or more medical PDFs from the sidebar
- Automatic text extraction, chunking, and embedding
- Semantic search over documents using FAISS
- Answers generated by an LLM grounded in your documents

## Tech Stack

- **Streamlit** – UI
- **LangChain** – text splitting, vector store, orchestration
- **FAISS** – vector similarity search
- **HuggingFace sentence-transformers** – embeddings (`all-mpnet-base-v2`)
- **Groq** – chat model (`llama-3.3-70b-versatile`)
- **pypdf** – PDF text extraction

## Setup

### 1. Clone the repo

```bash
git clone <your-repo-url>
cd medchatbot
```

### 2. Create a virtual environment and install dependencies

Using `uv`:
```bash
uv sync
```

Or using `pip` (with a manual `venv`):
```bash
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e .
```

> Install `uv` from [https://docs.astral.sh/uv/](https://docs.astral.sh/uv/) if you don't have it.

### 3. Add your API key

Copy the example file and fill in your Groq API key:
```bash
cp .env.example .env
```

Then edit `.env`:
```
GROQ_API_KEY=gsk-your-actual-key
```

Get a free key at [https://console.groq.com/keys](https://console.groq.com/keys).

## Run

```bash
streamlit run main.py
```

Open the URL Streamlit prints (usually `http://localhost:8501`), upload PDFs from the sidebar, click **Process Documents**, and start asking questions.

## How It Works

1. **Extract** – Text is pulled out of each uploaded PDF with `pypdf`.
2. **Chunk** – Text is split into overlapping chunks using `RecursiveCharacterTextSplitter`.
3. **Embed** – Each chunk is converted to a vector using a sentence-transformer model.
4. **Index** – Vectors are stored in a FAISS index (in-memory).
5. **Retrieve** – At query time, the top-k most similar chunks are retrieved.
6. **Answer** – The retrieved chunks are passed to the LLM as context, and it produces a grounded answer.

## Notes

- The FAISS index is in-memory and rebuilt on every "Process Documents" click — it is not persisted between sessions.
- The PDFs in `sample_data/` are **synthetic** and intended only for trying out the app.
- This is a learning project, not a medical tool. Do not rely on its outputs for real clinical decisions.
